Our calculator is capable of performing the four basic arithmetic operations: addition, subtraction, multiplication, and division. The result of these operations is displayed without clearing the input field, allowing users to retain and review the original expression. The application properly handles division by zero by displaying an error message and returning an undefined value (NaN). It also supports negative numbers.

The calculator supports three different notations: infix, prefix, and postfix notation. Infix notation is the most commonly used form, where the operator is placed between operands (e.g., 6 × 7). Prefix notation, used in languages such as Scheme, places the operator before the operands (e.g., * 3 9). Finally, postfix notation, used in programming languages such as Forth, places the operator after the operands (e.g., 0 8 −).

The calculator is capable of handling compound expressions, including those involving multiple operands or multiple operations. In prefix and postfix notations, it is possible to use a single operator with several operands (e.g., + 0 1 2 3 4 and 5 6 7 8 9 ×). The application also supports a more advanced notation. For instance, the infix expression (4 + 5 + 6) × (7 + (5 − 2 − 7)) × 9 can be rewritten as ((4 5 6)+ (7 (5 2 7)−)+ 9). In this case, the operation applied to the resulting expression (15 3 9) is multiplication, yielding 15 × 3 × 9 = 405. As in standard arithmetic, parentheses are used to enforce operation precedence, but only in infix notation.

The user interface provides several buttons for entering expressions, including both numbers and operators. A “clear” button allows the user to reset the input field entirely. A “delete” button removes a character based on the current cursor position. A “compute” button evaluates the expression entered in the input field.
