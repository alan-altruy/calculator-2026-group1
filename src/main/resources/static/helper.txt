Our calculator is capable of performing the four basic arithmetic operations: addition, subtraction, multiplication, and division. The result of these operations is displayed without clearing the input field, allowing users to retain and review the original expression. The application properly handles division by zero by displaying an error message and returning an undefined value (NaN). It supports negative numbers as well as several numeric types, including integers, rational numbers, irrational real numbers, and complex numbers.

The calculator supports three different notations: infix, prefix, and postfix notation. Infix notation is the most commonly used form, where the operator is placed between operands (e.g., 6 × 7). Prefix notation, used in languages such as Scheme, places the operator before the operands (e.g., * 3 9). Finally, postfix notation, used in programming languages such as Forth, places the operator after the operands (e.g., 0 8 −).

The calculator is capable of handling compound expressions, including those involving multiple operands or multiple operations. In prefix and postfix notations, it is possible to use a single operator with several operands (e.g., + 0 1 2 3 4 and 5 6 7 8 9 ×). The application also supports a more advanced notation. For instance, the infix expression (4 + 5 + 6) × (7 + (5 − 2 − 7)) × 9 can be rewritten as ((4 5 6)+ (7 (5 2 7)−)+ 9). In this case, the operation applied to the resulting expression (15 3 9) is multiplication, yielding 15 × 3 × 9 = 405. As in standard arithmetic, parentheses are used to enforce operation precedence, but only in infix notation.

Several unary operations and mathematical functions are available, including natural logarithm (ln), decimal logarithm (log), absolute value, factorial, and trigonometric functions such as sin, cos, tan, together with their inverse functions and hyperbolic functions. Unary operations can be written either directly before a value (for example, ln4, sin30, or 5!) or applied to a parenthesized expression (for example, ln(4), sin(30), or (3 + 2)!). Absolute value is written using vertical bars, such as |x|, following the usual mathematical convention. Scientific notation uses the "e" preceded by a real and followed by an integer (for example 54.3e-4). This notation allows unary operators to be used naturally while remaining compatible with more complex expressions.

Complex numbers must be written explicitly in the form a + b * i, where a represents the real part and b the imaginary coefficient. In other words, multiplication by i must always be written explicitly using the multiplication operator. For example, a complex number must be entered as 3 + 2 * i rather than 3 + 2i. This rule ensures that expressions remain unambiguous and consistent with the parser used by the application.

The application generates a random number within the specified range based on the provided seed. The calculator allows users to set the seed for the random number generator, enabling the generation of reproducible sequences of random numbers. In integer and rational mode, the user must provide the upper bound (for example random(1830) or random ), while in real and complex mode, the application generate a number between 0 and 1. The user have to write "random" in the input field.

The user interface provides several buttons for entering expressions, including both numbers and operators. A “clear” button allows the user to reset the input field entirely. A “delete” button removes a character based on the current cursor position. A “compute” button evaluates the expression entered in the input field.

Finally, in the case of invalid expressions (such as ambiguous, incomplete, or syntactically incorrect inputs, including the use of unsupported characters), the application displays an error message and returns the value “NaN”.
